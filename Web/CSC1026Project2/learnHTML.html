There are 5 key components to web architecture:
HTTP (HyperText Transfer Protocol)
URI (Uniform Resource Identifier)
HTML (HyperText Markup Language)
DNS (Domain Name System)
TCP/IP (Internet Protocol Suite)

HTTP is the protocol used to define how messages are transmitted, formatted and also what to do with certain parts.
When browsing the web, you will likely notice that the URL for the pages start with http:// (https:// in some cases).

HTTP headers are metadata for the request.
This metadata includes information such as Date, Length, Server, Cache-Control etc.

These are the two mandatory HTTP methods:
HEAD - Gets the headers for the page without returning any content (metadata).
GET - Gets the content of the requested page along with the metadata.

HTTP requests come in the format of:
Scheme - Authority - Path - Query - Fragment
https: //www.google.com
https: //authority.com /path /?name=dog #frag
Not all the fields are required. For example, Youtube's homepage is https://www.youtube.com
but a video link includes a query such as https://www.youtube.com/watch?v=PHgc8Q6qTjc

When you type in the address, the DNS server retrieves the IP address.
The browser then uses the IP address to connect to the web server.
Then your browser sends the HTTP request and the server responds with the website

When making a website, you should try to split the code into the different types.
HTML (HyperText Markup Language) is used for the content and structure so it goes into one file (or multiple for different pages)
CSS (Cascading Style Sheet) is mainly for the design and style of the page, so that goes into a separate file.
JavaScript is for interactivity and other scripts, so it goes into its own file too.
CSS and JavaScript can be split into its own smaller files too to help organise things better.

There are 3 main parts to HTML.
The content is the text, images, video and other stuff that goes onto the page.
The elements or tags is the structure for the content and focuses on organisation.
The attributes are special additions to the elements to add more information, like an ID.
A tag looks like this <html> (we will learn more about the different tags and their meanings later)

There are a few main versions of HTML that are still used today.
HTML is sort of the core of it all. HTML (and its different versions) have evolved but HTML 4 is quite common
and supports most things required for a basic site. XHTML is based heavily off HTML 4 and shares the same tags and attributes as it.
Only, XHTML was tightened up in terms of syntax. E.g tags and attributes must be lower case.
HTML 5 is the newest version and it supports a lot more than the other versions as it evolved, but got rid of some older tags and attributes. (so be careful)

The basic structure of a webpage is:
A line at the top containing HTML version information
A header section containing other information that is not displayed on the pages
A body section which contains all the other stuff that is to be shown on the page

The line containing HTML information is what tells the browser how it is supposed to translate the HTML.
It looks like this:
<!DOCTYPE HTML PUBLIC "‐//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
The first part says its a html 4.01 document and to use the W3C DTD (Document Type Definition).
The second part specifies where the DTD can be found.
This is basically the same for all pages, with a few differences depending on what you need.

This DTD is important because it enables the web browser to show the HTML how the creator intended.
It also means that it can be validated using online checkers to make sure it follows the rules of XHTML, or HTML 4 or which ever version chosen.

The first line of the document should always be the version information
For an XHTML document, it looks like this:
<!DOCTYPE HTML PUBLIC "‐//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1‐strict.dtd">
However, for HTML 5, this is what it might look like:
<!DOCTYPE html>

The HTML tags come next and they hold all of the other stuff in the Document
They look like this:
<html></html>
As you can see, one of them is different. This is because one opens the HTML section <html> and the other closes it </html>

Inside the HTML tags in the head tags. These contain stuff like the title, links to other files for CSS and JavaScript etc.
None of the tags in the head tag are displayed in the page.
The head tags look like this
<head></head>

The title tags belong in the head tags and they specify the title on the page.
This is shows at the very top inside the browser tab.
The one on this page should be: Making A Website! - Learn HTML
The title tags look like this:
<title></title>

The link tag is used to link external CSS and JavaScript files to the current HTML page
The link tag looks like this:
<link></link>
An example may look like this:
<link rel="stylesheet" type="text/css" href="css/navigation.css">
This links a CSS page called navigation

The body tags come after the closing tag of the head and contains the content.
The body tags themselves don't show up in the page but the stuff inside does.
The body tags look like this:
<body></body>

Inside the body tags, there are couple of different classifications of tags.
First is block level tags and they help to provide more structure to the page.
For example headings and horizontal rules are block level

Div tags are used to group together other block level tags. They become more useful the larger a page is.
They help to keep things organised and neat.
Div tags look like this:
<div></div>

Heading tags are next and they create visible heading on the page, like the Learning HTML title at the top of this page.
There are different levels of headings and it is important to follow the level order.
This means you should not skip the heading by going straight to level 3.
You are also not able to next them (place inside each other).
The different levels are as follows:
<h1></h1>, <h2></h2>, <h3></h3>, <h4></h4>, <h5></h5>, <h6></h6>

Paragraph tags are what is used to hold all the content for the page.
All the text you see will likely be in a paragraph tag, including this text and the text above.
Paragraph tags look like this:
<p></p>

Lists are next and they are useful for multiple reasons.
Lists can be used to create a numbered or bullet point set of information.
Lists can also be used to create the navigation bars at the top of pages, with the right formatting.
To create a numbered list, you use: <ol><li></li></ol>
To create a bullet point list, you use: <ul><li></li></ul>
The li tags inside are used to specify each item, so you open and close a new one for each new item.

Some other useful tags are <hr/> and <br/>. These tags self close as there is no content to be put inside them.
The <hr/> tag creates a horizontal line across the page and is used to separate content.
The <br/> tag is used to create a new line. This is used in paragraphs to separate the content.

Inline tags are used inside block tags, usually in paragraphs and allow for more customisability of specific sections.
For example, to make a specific work italic or bold, you would use inline tags.
This is because if you made the p tag bold, all the text would be bold, instead of the section you want.

Span tags are the inline equivalent of the div tags, which means that you can create a block and change it in specific way.
The span tags look like this:
<span></span>

To create italic text, you need to use the <em> tags.
This will set the text between the open and close to be italic.
These are used inline, most commonly inside the p tags.
The italic tags look like this:
<em></em>

To create bold text, you need to use the <strong> tags.
This will set the text between the open and close to be bold.
The bold tags look like this:
<strong></strong>

Code tags are used to display sections of code on a page.
This sets the text to a monospaced font and create a difference between the rest of the text.
The code tags look like this:
<code></code>

The acronym tag allow you to use the abbreviated form of a phrase,
while also allowing the user to hover over the acronym to see the full phrase.
This tag displays the text in between the tags but requires a title attribute to display when hovered.
The acronym tag looks like this:
<acronym title=""></acronym>

The link tags is used to send users to another site or page.
Once you specify the location to go to, once it is clicked, it will send the user there.
This requires a href tag to specify the location.
The link tags look like this:
<a href=""></a>

Each tag has its own attributes that it can be used with and they provide addition information to the tag.
The most common are class and id as these are used with pretty much all tags.
The attribute goes after specifying what tag it is but before the closing of the open tag.
<div class=""></div>

The class attribute gives the tag a class.
This means that when writing CSS, a specified style will be applied.
The class attribute can be used as many times in a html document.
The class attribute looks like this:
class=""

The id attribute is similar to class in that it is used to give an identifier to the tag.
The main difference is that you can only have one of a specified id in the html document,
meaning all ids must be unique.
The is attribute looks like this:
id=""

The image tag is used to add images to the page.
The image tag uses a src attribute to specify where the image to be used is.
The image tag can be styled using CSS but if not, it will be the original size of the image to be careful.
The image tag looks like this:
<img src=""></img>

There are a few different types of video tags, each requiring specific information.
IFrame is the most common for embeding youtube videos as that is the format youtube gives the embed code in.
The different types of tags are:
<embed></embed>, <object></object>, <video></video>, <iframe></iframe>
